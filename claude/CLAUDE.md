# CLAUDE.md

## チャットのルール

私は日本人のソフトウェア開発者です。
そのため、チャットでは日本語でのやりとりを希望します。

### 日本語の文章を生成する際の注意事項

これは強い要望です。

**体言止めを乱用しないでください！**
**特に、無理に動詞の名詞形で終わる文は非常に読みづらいため、動詞を省略せずに文を結んでください！**

また、箇条書きでは体言止めの文が多用されることが多い印象があります。
箇条書きの際には、特に注意してください。

ただし、以下のような例外的なケースでは、止むを得ず体言止めを使用することを許可します。

- 名詞で自然に結ばれる文
- コードコメントなど文章量が限られるケース

このドキュメント自体が、体言止めを注意深く使用しているので、参考にしてください。

### 質問に対して回答する際の注意事項

必ず、質問された内容を復唱してから答えます。
回答の最後には、その回答の正確性を%表記で出力します。
評価は厳正かつ正確に行います。

### 作業をする際の注意事項

ファイルの編集など、実際の作業を行う際は、私が確認できるように実行計画を提示してください。

### どこからどこまでが回答なのかを視覚的に表示する

回答が長い場合、ターミナル上での私の質問と回答の区別が難しいです。
そのため、回答のはじめとおわりに、赤色で以下の文章を表示してください。

```md
🍎🍎🍎 🤖 START 🤖 🍎🍎🍎

（回答内容）

🍏🍏🍏 🤖  END  🤖 🍏🍏🍏
```

## プロジェクトの編集ルール

- AI コーディングエージェントにまだ不慣れなので、git commit はしないでください。

## 一時ファイル使用時のルール

確認・検証・デバッグなどで一時的なファイルを使う時はリポジトリの配下に `tmp/ai-coding-agent/` ディレクトリを作って、そこを使って行うようにしてください。

また、質問の回答によっては、ファイルとして保存してあとから参照できるようにした方が好ましい場合があります。
例えば、不慣れなプログラミング言語の簡単なリファレンスなどが該当します。
こういったものも、同ディレクトリ内に保存するようにしてください。

## 禁止事項

- パスワードや API キーをハードコーディングしてはいけない
- ユーザーの確認なしにデータを削除してはいけない
- 起動したディレクトリ配下のファイル以外は read してはいけない

## 必須事項

- 関数、クラス、型などには、言語ごとの慣習に沿ったドキュメンテーションコメントを付与する
    - 例: JavaScript における JSDoc、Python における docstring など
- エラーハンドリングを実装する
- 変更前に既存テストが通ることを確認する

## 重要事項

- パフォーマンスへの影響を考慮する
- 後方互換性を維持する
- セキュリティベストプラクティスに従う

## プログラミング言語ごとのルール

### TypeScript

- `any` や `as` を極力避けること
    - テストコードなどで、`any` を使わないことによって大幅に記述量が増える場合などは例外とする
- `unknown` は使っても良い

### Go

- ソースファイル編集後に、`goimports` コマンドによってソースファイルをフォーマットすること
