# CLAUDE.md

## チャットのルール

私は日本人のソフトウェア開発者です。
そのため、チャットでは日本語でのやりとりを希望します。

### 質問に対して回答する際の注意事項

必ず、質問された内容を復唱してから答えます。
回答の最後には、その回答の正確性を `%` 表記で出力します。
評価は厳正かつ、客観的に行います。

### 作業をする際の注意事項

ファイルの編集など、実際の作業を行う際は、私が確認できるように実行計画を提示してください。

### どこからどこまでが回答なのかを視覚的に表示する

回答が長い場合、ターミナル上での私の質問と回答の区別が難しいです。
そのため、回答のはじめとおわりに、以下のようなマーカーを表示してください。

```md
🚀🚀🚀 🤖 START 🤖 🚀🚀🚀

（回答内容）

✅✅✅ 🤖  END  🤖 ✅✅✅
```

## プロジェクトの編集ルール

### Git のルール

- git コマンドについては、使用する際に確認を求めてください。
- `git commit` に関して、以下に注意してください。
    - コミットは、レビューしやすい適切な粒度で作ってください
    - conventional commits を意識して、コミットメッセージを書いてください
- **force push は禁止です！**

### 一時ファイル使用時のルール

確認・検証・デバッグなどで一時的なファイルを使う時はリポジトリの配下に `__tmp/ai-coding-agent/` ディレクトリを作って、そこを使って行うようにしてください。

また、質問の回答によっては、ファイルとして保存してあとから参照できるようにした方が好ましい場合があります。
例えば、不慣れなプログラミング言語の簡単なリファレンスなどが該当します。
こういったものも、同ディレクトリ内に保存するようにしてください。

## 禁止事項

- パスワードや API キーをハードコーディングしてはいけない
- ユーザーの確認なしにデータを削除してはいけない
- 起動したディレクトリ配下のファイル以外は read してはいけない

## 必須事項

- 関数、クラス、型などには、言語ごとの慣習に沿ったドキュメンテーションコメントを付与する
    - 例: JavaScript における JSDoc、Python における docstring など
- エラーハンドリングを実装する
- 変更前に既存テストが通ることを確認する

## 重要事項

- パフォーマンスへの影響を考慮する
- 後方互換性を維持する
- セキュリティベストプラクティスに従う
